Regularization Paths

Track how model coefficients change as regularization strength varies.

Why Regularization Paths?

In convex learning models, the regularization parameter \(\lambda\) controls bias-variance tradeoff, sparsity, and stability. A path view shows how solutions evolve continuously as \(\lambda\) changes.

\[\min_w\; \mathcal{L}(w) + \lambda \mathcal{R}(w).\]

Typical Behaviors

L1 (Lasso)

As \(\lambda\) grows, many coefficients hit exactly zero (sparsity).

L2 (Ridge)

Coefficients shrink smoothly toward zero, rarely exactly zero.

Elastic Net

Interpolates between sparsity and smooth shrinkage.

Practical Checklist

  • Sweep \(\lambda\) on a log scale, not linear only.
  • Track validation error alongside coefficient magnitude.
  • For sparse modeling, monitor number of active coefficients.
  • Use warm starts when solving along a path for speed.
