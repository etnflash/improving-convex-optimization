Feasibility and Projection Algorithms (POCS / Dykstra)

Find points in intersections of convex sets using repeated projections.

Convex Feasibility Problem

Instead of minimizing an objective, feasibility seeks

\[\text{find } x \in C_1 \cap C_2 \cap \cdots \cap C_m.\]

Projection-based methods are simple and scalable when each set has an easy projector.

POCS: Alternating Projections

For two sets \(A\) and \(B\), POCS iterates

\[x^{k+1}=\Pi_B(\Pi_A(x^k)).\]

If \(A\cap B\neq\emptyset\), the sequence approaches a feasible point under standard assumptions. 

repeat
    x <- projection_A(x)
    x <- projection_B(x)
until feasibility residual is small

Dykstra's Algorithm

Dykstra adds correction vectors to recover the projection onto the intersection itself, not just any feasible limit point. For two sets:

\[\begin{aligned} y^{k+1} &= \Pi_A(x^k + p^k),\quad p^{k+1}=x^k + p^k - y^{k+1},\\ x^{k+1} &= \Pi_B(y^{k+1}+q^k),\quad q^{k+1}=y^{k+1}+q^k-x^{k+1}. \end{aligned}\]
