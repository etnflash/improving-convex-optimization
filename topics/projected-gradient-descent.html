Projected Gradient Descent

Handle convex constraints by alternating gradient steps with projection back to the feasible set.

Constrained Problem Setup

\[\min_{x \in C} f(x),\]

where \(f\) is convex and differentiable, and \(C\) is a closed convex feasible set. A plain gradient step can leave \(C\), so we correct it with projection.

Core Update Rule

Define Euclidean projection

\[\Pi_C(v)=\arg\min_{x\in C} \|x-v\|_2.\]

Projected gradient descent (PGD) iteration:

\[x_{k+1}=\Pi_C\left(x_k-\alpha_k\nabla f(x_k)\right).\] 

repeat
    y_k = x_k - alpha_k * grad_f(x_k)
    x_{k+1} = projection_C(y_k)
until stopping criterion

Geometric Intuition

The gradient step improves the objective locally. Projection then picks the closest feasible point, preserving progress while respecting constraints. For convex \(C\), this projection is unique.

Example: Box Constraints

If \(C=[-\rho,\rho]^n\), projection is coordinate-wise clipping:

\[\Pi_C(v)_i = \max(-\rho,\min(\rho,v_i)).\]

Practical Benefit

Cheap projection enables scalable optimization for large-dimensional bounded models.

Step-size Choices

  • Fixed \(\alpha\): works with known smoothness constants.
  • Backtracking line search: adaptive and robust in practice.
  • Diminishing \(\alpha_k\): common in subgradient-like settings.

Convergence Snapshot

For convex \(f\) with \(L\)-Lipschitz gradient and constant \(\alpha \in (0,2/L)\), PGD converges. Strong convexity gives linear rates under standard assumptions.
